# Student Portal — Spring Boot Application

A role-based Student Portal system built with Spring Boot, Spring Security (JWT), JPA/Hibernate, and MySQL. Manages Users (Admin, Teacher, Student), Classes, Subjects, Exams, and Marks with strict role-based access control.

## Prerequisites

- **Java 17** or higher
- **MySQL 8.0** or higher

## Database Setup

Connect to your MySQL server and run:

```sql
CREATE DATABASE manulife_student_portal;
```

The application uses the following default database configuration:

| Property | Value |
|----------|-------|
| URL | `jdbc:mysql://localhost:3306/manulife_student_portal` |
| Username | `root` |
| Password | `mysql` |

> If your MySQL credentials differ, update them in `src/main/resources/application.yaml` before building, or directly in the JAR's configuration.

All tables are auto-generated by Hibernate on application startup (`ddl-auto: update`). No manual schema import is needed.

## Running the Application

```bash
java -jar student-portal.jar
```

The application starts on **port 8081**.

## Default Admin Credentials

A default admin user is auto-created on first startup:

| Field | Value |
|-------|-------|
| Username | `admin` |
| Password | `admin123` |

## API Documentation (Swagger)

Once the application is running, Swagger UI is available at:

```
http://localhost:8081/swagger-ui/index.html
```

Use the **Authorize** button in Swagger UI to enter your JWT token (obtained from the login endpoint) for testing protected APIs.

## Postman Collection

A Postman collection is included in the submission (`Student_Portal_API.postman_collection.json`). It contains all API endpoints with sample request/response bodies and authorization token examples.

## Tech Stack

- Java 17, Spring Boot 3.3.5

- Spring Data JPA / Hibernate 6
